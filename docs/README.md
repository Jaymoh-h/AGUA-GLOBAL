## Documentation Principles

- Keep docs aligned with code before major releases.
- Add implementation notes when migrations, billing rules, payment behavior, or role permissions change.
- Treat workflows as business procedures, not just technical descriptions.
- Mark future work clearly so readers can tell implemented capability from planned expansion.
